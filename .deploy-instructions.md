## 🚀 OPCIÓN 1: Despliegue vía Git (Recomendado)

### Paso 1: Commit de cambios locales

```bash
# Verificar estado
git status

# Agregar solo el archivo modificado del backend
git add backend/pagoCuotas/views.py

# Crear commit con descripción clara
git commit -m "Fix: Corrección de filtrado de cuotas por estado de contrato

- Implementado Exists() para verificar correctamente pares (RUT, Patente)
- Solucionado problema donde no aparecían cuotas de contratos activos
- Corregida lógica en PagoCuotasAPIView y CuotasImpagasAPIView"

# Verificar que el commit se creó correctamente
git log -1
```

### Paso 2: Subir a repositorio remoto

```bash
# Push a la rama main (o la rama que uses)
git push origin main
```

### Paso 3: Actualizar en el servidor cPanel

**Conéctate vía SSH a tu servidor cPanel:**

```bash
ssh usuario@tu-dominio.com
```

**O si usas Git Deployment en cPanel:**
1. Ir a cPanel → Git™ Version Control
2. Buscar tu repositorio
3. Click en "Pull or Deploy" → "Update from Remote"

**O manualmente vía SSH:**

```bash
# Navegar al directorio del proyecto
cd ~/ruta-del-proyecto

# Pull de los cambios
git pull origin main

# Reiniciar el servidor Python (según tu configuración)
# Opción A - Si usas passenger_wsgi.py:
touch tmp/restart.txt

# Opción B - Si tienes un proceso Python corriendo:
pkill -f "python manage.py"
python manage.py runserver --host 0.0.0.0

# Opción C - Si usas systemctl:
sudo systemctl restart nombre-del-servicio
```

## 🚀 OPCIÓN 2: Despliegue Manual vía FTP/SFTP

### Paso 1: Conectar vía FTP

Usa FileZilla o WinSCP con estas credenciales:
- **Host:** ftp.tu-dominio.com (o la IP de tu servidor)
- **Usuario:** tu-usuario-cpanel
- **Contraseña:** tu-contraseña
- **Puerto:** 21 (FTP) o 22 (SFTP)

### Paso 2: Subir archivo

1. Navega a la ruta en el servidor: `/home/usuario/ruta-del-proyecto/backend/pagoCuotas/`
2. **IMPORTANTE:** Haz un backup del archivo actual:
   - Descarga `views.py` y guárdalo como `views.py.backup-2026-01-16`
3. Sube el nuevo archivo `views.py` desde:
   - Local: `d:\automotriz\backend\pagoCuotas\views.py`
   - Destino: `/home/usuario/ruta-del-proyecto/backend/pagoCuotas/views.py`

### Paso 3: Reiniciar el servidor

Desde cPanel o SSH:
```bash
touch ~/ruta-del-proyecto/tmp/restart.txt
```

## 🚀 OPCIÓN 3: Despliegue vía cPanel File Manager

### Paso 1: Backup del archivo actual

1. Ir a cPanel → File Manager
2. Navegar a: `/home/usuario/ruta-del-proyecto/backend/pagoCuotas/`
3. Click derecho en `views.py` → Copy
4. Renombrar la copia a `views.py.backup-2026-01-16`

### Paso 2: Subir nuevo archivo

1. Click en "Upload" en File Manager
2. Seleccionar: `d:\automotriz\backend\pagoCuotas\views.py`
3. Sobrescribir el archivo existente

### Paso 3: Reiniciar aplicación

1. Ir a cPanel → Setup Python App (o similar)
2. Click en "Restart" en tu aplicación

## 🐛 Troubleshooting

### Problema: "Failed to fetch" en el frontend

**Solución:**
```bash
# Verificar que el servidor está corriendo
ps aux | grep python

# Revisar logs por errores
tail -50 ~/logs/django.log
```

### Problema: Aún no aparecen las cuotas

**Solución:**
1. Verificar que el archivo se subió correctamente
2. Confirmar que el servidor se reinició
3. Limpiar caché del navegador (Ctrl+Shift+R)
4. Revisar logs del backend para ver mensajes DEBUG
